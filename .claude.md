# PolicyPulse — Shared Development Guide

## Project Overview

PolicyPulse is a personal policy impact advisor for HooHacks 2026. Users ask how a policy affects *them* and a multi-agent AI system researches sources, pays for premium data via Lightning micropayments (L402), and delivers a personalized impact analysis.

**Target tracks:** Best AI, Best Data Science, Capital One, Deloitte, SZNS Solutions

## Tech Decisions

- **Backend:** FastAPI (Python 3.11+), async everywhere
- **Database:** PostgreSQL + pgvector (HNSW indexing)
- **LLM calls:** OpenAI SDK (GPT-4) or Google GenAI SDK (Gemini) — use structured JSON output where possible
- **Agents:** Start with simple async orchestration (`asyncio.gather` for parallel sector agents). Upgrade to CrewAI/LangGraph only if time permits
- **RAG:** 20-30 pre-embedded docs per demo scenario, hybrid retrieval
- **Lightning:** LND on testnet, real L402 protocol flow
- **Frontend:** Next.js 14+ (App Router), Tailwind CSS, D3.js for Sankey
- **Streaming:** Server-Sent Events (SSE) from backend to frontend

## API Contract

The main interface between frontend and backend:

### `POST /api/query`
```json
{
  "question": "How will the new H1B visa policy affect international students graduating in 2026?",
  "context": {
    "role": "international_student",
    "graduation_year": 2026,
    "field": "computer_science",
    "concerns": ["opt", "visa_sponsorship"]
  }
}
```

Response: SSE stream of agent events.

### SSE Event Format
```json
{
  "type": "agent_start" | "agent_result" | "lightning_payment" | "final_result" | "error",
  "agent": "analyst" | "premium" | "sector_jobs" | "sector_legal" | "sector_financial" | "sector_historical" | "synthesis" | "debate",
  "data": { ... },
  "timestamp": "ISO8601"
}
```

### `lightning_payment` event
```json
{
  "type": "lightning_payment",
  "data": {
    "service": "premium-legal-db",
    "invoice_amount_sats": 10,
    "status": "paid",
    "macaroon_received": true
  }
}
```

### `final_result` shape
```json
{
  "summary": "...",
  "claims": [
    {
      "text": "...",
      "confidence": "high" | "medium" | "low",
      "sources": [
        { "title": "...", "url": "...", "type": "free" | "premium" }
      ]
    }
  ],
  "sankey_data": { "nodes": [...], "links": [...] },
  "agent_trace": [...]
}
```

These schemas are a starting point — iterate freely, but update this file when you change them so the other side of the interface stays in sync.

## Agent Pipeline

```
User Query
    │
    ▼
┌─────────┐
│ Analyst  │ ── RAG retrieval (public sources)
└────┬────┘
     │
     ▼
┌──────────┐
│ Premium  │ ── L402 payment → premium data access
└────┬─────┘
     │
     ▼ (parallel)
┌──────────┐ ┌──────────┐ ┌───────────┐ ┌────────────┐
│ Sector:  │ │ Sector:  │ │ Sector:   │ │ Sector:    │
│ Jobs     │ │ Legal    │ │ Financial │ │ Historical │
└────┬─────┘ └────┬─────┘ └─────┬─────┘ └─────┬──────┘
     │             │             │              │
     └─────────────┴─────────┬───┴──────────────┘
                             ▼
                      ┌────────────┐
                      │ Synthesis  │ ── aggregate + confidence score
                      └─────┬──────┘
                            ▼
                      ┌────────────┐
                      │  Debate    │ ── one round of adversarial challenge
                      └─────┬──────┘
                            ▼
                       Final Result
```

Each agent is an async function that takes context and returns structured output. Keep them independently testable.

## Development Conventions

- **Python:** Use type hints. Pydantic models for all data flowing between agents. `async def` for anything that does I/O.
- **Frontend:** TypeScript strict mode. Components get their own file. Keep state simple (React state, no Redux).
- **Environment:** All secrets and config in `.env` (never committed). Use `.env.example` as the template.
- **Commits:** Short descriptive messages. Prefix with module area: `backend:`, `frontend:`, `lightning:`, `infra:`, `rag:`.
- **Branches:** Work on feature branches, merge to `main` when stable. Don't block each other.

## What's Real vs. Mocked

**Must be real:**
- LLM-generated personalized responses (not canned text)
- At least one real L402 Lightning payment on testnet
- Real-time agent activity feed showing actual pipeline steps

**Simplified but functional:**
- RAG over small curated corpus (not the whole internet)
- Sector agents as parallel LLM calls with different system prompts
- Single-round Debate Agent

**Mocked / pre-computed for demo safety:**
- Sankey diagram data pre-computed for the 3 demo scenarios, mapped to agent output
- PDF report export uses pre-styled template
- User context in React state only (no DB, no auth)
- Cached fallback responses for demo scenarios in case of API failures

## Quick Start

```bash
# Clone and setup
cp .env.example .env
# Fill in API keys (OpenAI/Gemini, LND connection)

# Docker (full stack)
docker compose up --build

# Backend only (dev)
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend only (dev)
cd frontend
npm install
npm run dev
```
